# پنل مدیریت کاربران و کانفیگ

اپلیکیشن وب کامل (بک‌اند Node.js/Express + دیتابیس SQLite + فرانت‌اند) برای مدیریت کاربران و تولید کانفیگ اتصال (VLESS / VMess).

## پیش‌نیاز
- Node.js نسخه ۱۸ به بالا

## نصب و اجرا

```bash
cd config-panel
npm install
cp .env.example .env
```

فایل `.env` رو باز کن و این مقادیر رو عوض کن:
- `JWT_SECRET`: یک رشته‌ی تصادفی و طولانی
- `ADMIN_USERNAME` / `ADMIN_PASSWORD`: مشخصات ورود اولیه‌ی ادمین (فقط در اولین اجرا استفاده می‌شه و در دیتابیس هش می‌شه)

بعد اجرا کن:

```bash
npm start
```

پنل روی آدرس `http://localhost:3000` بالا میاد (یا پورتی که در `.env` تنظیم کردی).

## دیپلوی روی سرور واقعی
- می‌تونی با `pm2` پروسه رو دائمی نگه داری: `pm2 start server.js --name config-panel`
- پشت Nginx بذارش و با TLS (Let's Encrypt / Certbot) روی دامنه‌ی خودت سرو کن
- فایل `data.sqlite` همون دیتابیسه — بکاپ‌گیری منظم ازش رو فراموش نکن

## نکته‌ی مهم درباره‌ی کانفیگ‌ها
این پنل فقط **رشته‌ی اتصال (کانفیگ)** رو بر اساس تنظیمات سرور و UUID هر کاربر می‌سازه؛ خودش سرور Xray/V2Ray رو اجرا یا مدیریت نمی‌کنه. یعنی:
- باید یک سرور Xray/V2Ray واقعی جدا داشته باشی که روش اجرا شده
- UUID هر کاربر رو باید در کانفیگ سمت سرور (inbound) هم اضافه کنی تا واقعاً کار کنه
- اگه بخوای این دو تا به هم وصل بشن (یعنی پنل خودش inbound سرور Xray رو هم آپدیت کنه)، باید یک بخش اضافه بنویسیم که با API یا فایل کانفیگ Xray در تعامل باشه — بگو تا اون قسمت رو هم اضافه کنیم.

## ساختار پروژه
```
config-panel/
  server.js          # نقطه‌ی ورود Express
  db/index.js         # اتصال SQLite و schema
  routes/auth.js       # لاگین و تغییر رمز
  routes/users.js      # CRUD کاربران + تولید کانفیگ
  routes/settings.js   # تنظیمات سرور
  public/              # فرانت‌اند (HTML/CSS/JS ساده)
```
